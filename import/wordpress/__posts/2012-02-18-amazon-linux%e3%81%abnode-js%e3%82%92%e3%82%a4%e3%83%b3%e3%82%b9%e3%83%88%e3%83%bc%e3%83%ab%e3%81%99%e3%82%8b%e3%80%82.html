Twitter Streaming APIをブラウザ上で利用するために、Node.js+socket.ioを利用することにしました。そこで、Node.jsをAmazonEC2にインストールしました。以下、その際のメモです。

Node.js: http://nodejs.org/</a>

1. yum から必要なモジュールをインストール</strong>

  1. git</li>
  2. wget</li>
  3. gcc-c++</li>
  4. openssl-devel</li>
    </ol>

    $ sudo yum install wget git gcc-c++ openssl-devel</pre>
    

    なお、gcc-c++をインストールしていないと以下のような警告が出ました。</address>
    

    Checking for program g++ or c++          : not found
    
Checking for program icpc                : not found
    
Checking for program c++                 : not found
    
/home/ec2-user/.nvm/src/node-v0.4.11/wscript:232: error: could not configure a cxx compiler!</pre>
    
また、openssl-develをインストールしていないと以下のような警告が出ました。
    

    Checking for openssl : not found
    
Checking for function SSL_library_init : not found
    
Checking for header openssl/crypto.h : not found
    
/home/ec2-user/.nvm/src/node-v0.4.11/wscript:341: error: Could not autodetect OpenSSL support. Make sure OpenSSL development packages are installed. Use configure --without-ssl to disable this message.</pre>
    
</div>
    
2. nvmのソースコードをcheckout</strong>
    

    nvm (node version manager) のソースコードをgithubからダウンロード。実態はシェルスクリプトです。
    

    $ git clone git://github.com/creationix/nvm.git ~/.nvm</pre>
    
3. nvmを使う</strong>
    

    

    $ source ~/.nvm/nvm.sh</pre>
    
</div>
    
nvmを使うたびにこの .sh を読み込まなければいけないので、忘れないうちに、上記のコマンドを .bashrc に追加します。
    

    4. install node and use</strong>
    

    

    $ nvm install v0.4.11
    
$ nvm use v0.4.11
    
$ node --version
    
v0.4.11</pre>
